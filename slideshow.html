BPMN Process Flows

Process Flow 1: Development of Roles & Responsibilities (R&R) Documentation

Objective:

Achieve clarity on Roles and Responsibilities and redistribute/add work activities.

BPMN Diagram:

Swimlanes & Partners:

  • VEMO: Primary author and coordinator (Accountable/Responsible per Executive Summary).
  • MACs: Contributor/Reviewer (Consulted).
  • SAM (and ITOs): Contributor/Reviewer (Consulted).
  • Field Planning Board: Approver (implied governance body).

Start Event:

Mandate to clarify R&R based on the approved hybrid VEMO model (Field Planning Board, March 25, 2025).

Process Steps & Flows:

Swimlane: VEMO

  • Task: Initiate R&R documentation process. (Input: Approved proposal for the hybrid VEMO model)
  • Task: Draft Roles & Responsibilities document. (Output: Draft R&R Document)
  • Task: Distribute Draft R&R Document for review. (Output Message Flow: Draft R&R Document sent to MACs, SAM, and ITOs)

Swimlane: MACs (Responsibility: Provide comments/feedback - Consulted)

  • Input Message Flow: Draft R&R Document from VEMO
  • Task: Review Draft R&R Document.
  • Task: Provide comments/feedback. (Output: MACs Feedback on R&R Draft)
  • Task: Send MACs Feedback on R&R Draft to VEMO. (Output Message Flow)

Swimlane: SAM (and ITOs) (Responsibility: Provide comments/feedback - Consulted)

  • Input Message Flow: Draft R&R Document from VEMO
  • Task: Review Draft R&R Document.
  • Task: Provide comments/feedback. (Output: SAM/ITOs Feedback on R&R Draft)
  • Task: Send SAM/ITOs Feedback on R&R Draft to VEMO. (Output Message Flow)

Swimlane: VEMO

  • Input Message Flows: MACs Feedback on R&R Draft, SAM/ITOs Feedback on R&R Draft
  • Task: Consolidate feedback and revise document. (Output Updated: Draft R&R Document)
  • Task: Finalize Roles & Responsibilities Documentation. (Output: Final R&R Document)
  • Task: Submit Final R&R Document for approval by Field Planning Board. (Output Message Flow: Final R&R Document to Field Planning Board)

Swimlane: Field Planning Board

  • Task: Review and approve Final R&R Document. (Output: Approved R&R Document)

Swimlane: VEMO

  • Task: Disseminate Approved R&R Document to Product Teams and other stakeholders. (Output Message Flow)

End Event:

Final R&R Document approved by Field Planning Board and published.

Potential KPIs/Measures:

  • Timeliness of Final R&R Document completion (e.g., days from start to approval).
  • Completeness and clarity of the document based on stakeholder acceptance (e.g., survey score).
  • Number of review cycles required.
  • Stakeholder approval rate by Field Planning Board (e.g., percentage approved without revisions).

Process Flow 2: Development of Vendor/Licensing/Procurement Process Map/Roadmap

Objective:

Provide clarity on processes for teams/clients engaging in vendor, licensing, and procurement activities.

BPMN Diagram:

Swimlanes & Partners:

  • VEMO: Primary author and coordinator (Accountable/Responsible).
  • MACs: Subject Matter Expert, Contributor/Reviewer (Consulted).
  • SAM (and ITOs): Subject Matter Expert, Contributor/Reviewer (Consulted).
  • Product Teams: Informed, Recipient.

Start Event:

Need identified for clear process guidance for vendor/licensing/procurement activities.

Process Steps & Flows:

Swimlane: VEMO

  • Task: Initiate Process Map/Roadmap development.
  • Task: Request subject matter information. (Output Message Flow: Request for Process Subject Matter Information sent to MACs, SAM, ITOs)

Swimlane: MACs (Responsibility: Provide subject matter information and comments/feedback - Consulted)

  • Input Message Flow: Request for Process Subject Matter Information.
  • Task: Compile subject matter information on MACs’ procurement processes. (Output: MACs Process Information)
  • Task: Send MACs Process Information to VEMO. (Output Message Flow)

Swimlane: SAM (and ITOs) (Responsibility: Provide subject matter information and comments/feedback - Consulted)

  • Input Message Flow: Request for Process Subject Matter Information.
  • Task: Compile subject matter information on SAM/ITOs’ processes. (Output: SAM/ITOs Process Information)
  • Task: Send SAM/ITOs Process Information to VEMO. (Output Message Flow)

Swimlane: VEMO

  • Input Message Flows: MACs Process Information, SAM/ITOs Process Information.
  • Task: Draft Process Map/Roadmap. (Output: Draft Process Map/Roadmap)
  • Task: Distribute Draft Process Map/Roadmap for review. (Output Message Flow: Draft Process Map/Roadmap sent to MACs, SAM, ITOs)

Swimlane: MACs, SAM, ITOs (Parallel Review) (Responsibility: Provide comments/feedback - Consulted)

  • Input Message Flow: Draft Process Map/Roadmap.
  • Task: Review Draft Process Map/Roadmap.
  • Task: Provide comments/feedback. (Output: Feedback on Process Map Draft from each)
  • Task: Send Feedback on Process Map Draft to VEMO. (Output Message Flow)

Swimlane: VEMO

  • Input Message Flows: Feedback on Process Map Draft.
  • Task: Consolidate feedback and revise Process Map/Roadmap.
  • Task: Finalize Process Map/Roadmap. (Output: Final Process Map/Roadmap)
  • Task: Disseminate Final Process Map/Roadmap to Product Teams and clients. (Output Message Flow)

End Event:

Final Process Map/Roadmap approved and communicated to stakeholders.

Potential KPIs/Measures:

  • Timeliness of Final Process Map/Roadmap completion (e.g., days to completion).
  • Actionability and clarity of the roadmap for end-users (e.g., user survey score).
  • Reduction in process-related queries post-publication (e.g., percentage decrease in queries).
  • Stakeholder adoption rate (e.g., percentage of teams using the roadmap).

Process Flow 3: Creation of Vendor Profile Dashboard

Objective:

Increase data collection, reporting, and advice from VEMO functions.

BPMN Diagram:

Swimlanes & Partners:

  • VEMO: Owner, Developer, Maintainer (Accountable/Responsible per RACI).
  • MACs: Information Provider (Consulted, if applicable).
  • SAM (and ITOs): Information Provider (Consulted, if applicable).
  • Product Teams: Informed (per RACI).

Start Event:

Objective to increase data collection; planning launch for Summer 2025.

Process Steps & Flows:

Swimlane: VEMO

  • Task: Define requirements for Vendor Profile Dashboard focusing on STC Top 5 Vendors. (Input: Need for increased data collection, reporting, advice)
  • Task: Initiate creation of the Vendor Profile Dashboard.
  • Task (Conditional): Request subject matter information, if applicable. (Output Message Flow, Conditional: Request for Vendor Dashboard Data sent to MACs, SAM, ITOs)

Swimlane: MACs (Responsibility: Provide subject matter information, if applicable - Consulted)

  • Input Message Flow, Conditional: Request for Vendor Dashboard Data.
  • Task: Provide applicable information/data. (Output Data Object Created, Conditional: MACs Vendor Data for Dashboard)
  • Task: Send MACs Vendor Data for Dashboard to VEMO. (Output Message Flow, Conditional)

Swimlane: SAM (and ITOs) (Responsibility: Provide subject matter information, if applicable - Consulted)

  • Input Message Flow, Conditional: Request for Vendor Dashboard Data.
  • Task: Provide applicable information/data. (Output Data Object Created, Conditional: SAM/ITOs Vendor Data for Dashboard)
  • Task: Send SAM/ITOs Vendor Data for Dashboard to VEMO. (Output Message Flow, Conditional)

Swimlane: Product Teams (Responsibility: Informed on dashboard creation - per RACI)

  • Input (Implied): Updates from VEMO on dashboard progress.

Swimlane: VEMO

  • Input (Data Objects, Conditional): MACs Vendor Data for Dashboard, SAM/ITOs Vendor Data for Dashboard.
  • Task: Consolidate received data.
  • Task: Create and maintain the Vendor Profile Dashboard. (Output Data Object Created/Maintained: Vendor Profile Dashboard)
  • Task: Plan launch (Summer 2025).
  • Task: Disseminate Vendor Profile Dashboard access to stakeholders. (Output Message Flow: Dashboard access details to Product Teams and other stakeholders)

End Event:

Vendor Profile Dashboard launched (Summer 2025).

Potential KPIs/Measures:

  • Dashboard launch by Summer 2025 (e.g., on-time completion).
  • Accuracy and relevance of dashboard data (e.g., data error rate).
  • User adoption/usage of the dashboard post-launch (e.g., number of active users).
  • Data refresh frequency (e.g., updates per month).

Process Flow 4: Departmental Procurement Strategy Development, Execution & Alignment

Objective:

Execute STC and GC procurement strategies and align departmental procurement with overall direction.

BPMN Diagram:

Swimlanes & Partners:

  • VEMO: Strategy Contributor, Alignment Role (Responsible per RACI).
  • Product Teams (CCPD, ADCPD, DSPD): Requirements Definer, Strategy Contributor (Accountable/Responsible per RACI).
  • MACs: Procurement Strategy Definer, Execution Lead (Accountable/Responsible per RACI).
  • ITOS/SAM: Technical Consultant, Alignment Role (Responsible per RACI).

Start Event:

Business need from Product Team or strategic sourcing need identified by VEMO.

Process Steps & Flows:

Swimlane: VEMO (Responsibility: Develop and maintain Overall Vendor Management Strategy; align with STC procurement direction)

  • Task: Develop and maintain Overall Vendor Management Strategy. (Output: Overall Vendor Management Strategy)
  • Task: Proactively contribute to Vendor Roadmaps in partnership with Product Teams.
  • Task: Collaborate in defining departmental procurement strategies with MACs and Product Teams. (Input: Product/Service Requirements from Product Teams; Output Information Shared: Strategic vendor insights)

Swimlane: Product Teams (Responsibility: Define Product/Service Requirements; align with STC procurement direction)

  • Task: Define Product/Service Requirements. (Output: Product/Service Requirements)
  • Task: Contribute insights for departmental procurement strategy definition.

Swimlane: MACs (Responsibility: Execute departmental procurement strategy; align with STC procurement direction)

  • Task: Define STC procurement direction and departmental procurement strategy (in collaboration). (Input: Overall Vendor Management Strategy from VEMO, Product/Service Requirements from Product Teams; Output: Departmental Procurement Strategy)
  • Task: Execute Departmental Procurement Strategy (vendor identification, RFPs/RFIs, bid evaluations, negotiations). (Output: RFPs, Bid Evaluations, Contracts)
  • Task: Manage contracts, invoicing, compliance.

Swimlane: ITOS/SAM (Responsibility: Consulted on procurement execution; align with STC procurement direction)

  • Input Message Flow: Consultation Request from MACs for technical input.
  • Task: Provide technical input/impact assessment. (Output Message Flow: Technical Feedback to MACs)
  • Task: Ensure SAM activities align with procurement direction.

Swimlane: MACs

  • Task: Disseminate procurement outcomes to stakeholders. (Output Message Flow: Procurement outcomes to Product Teams, VEMO, ITOS/SAM)

End Event:

Contract awarded, service procured, managed, and aligned.

Potential KPIs/Measures:

  • Cycle time for procurement processes.
  • Cost savings/value realization.
  • Compliance rate with procurement policies.
  • Stakeholder satisfaction.
  • Alignment with GC procurement policies.

Process Flow 5: Vendor Issue Escalation Management

Objective:

Effectively manage and resolve escalated vendor support and interaction issues.

BPMN Diagram:

Swimlanes & Partners:

  • Product Teams: Issue Identifier, Escalation Accountable (A per RACI).
  • ITOS/SAM: Issue Resolution Responsible (R per RACI).
  • VEMO: Consulted or Accountable/Responsible for coordination if requested (C or A/R per RACI).
  • MACs: Informed (I per RACI).

Start Event:

Product Team encounters an unresolved vendor support/interaction issue.

Process Steps & Flows:

Swimlane: Product Teams (Responsibility: Vendor interaction; accountable for issue escalation)

  • Task: Identify vendor support/interaction issue.
  • Task: Attempt initial resolution.
  • Task (Gateway): If issue unresolved, escalate. (Output: Issue Escalation Report; Output Message Flow: Report to ITOS/SAM (R), VEMO (C), MACs (I))

Swimlane: ITOS/SAM (Responsibility: Responsible for issue escalation resolution)

  • Input Message Flow: Issue Escalation Report.
  • Task: Investigate escalated issue.
  • Task: Liaise with vendor for resolution.
  • Task (Gateway): If resolution requires further VEMO coordination for commercial vendors: Task: Request issue escalation support from VEMO. (Output Message Flow: Request for Escalation Support to VEMO)

Swimlane: VEMO (Responsibility: Consulted on issue escalation; A/R for coordinating commercial vendor engagement)

  • Input Message Flow: Issue Escalation Report OR Request for Escalation Support from ITOS/SAM.
  • Task (If support requested): Coordinate further engagement with commercial vendors (excluding SSC).
  • Task: Monitor and support issue resolution.

Swimlane: MACs (Responsibility: Informed on issue escalation)

  • Input Message Flow: Issue Escalation Report.

Swimlane: ITOS/SAM

  • Task: Document resolution and disseminate outcomes to stakeholders. (Output Message Flow: Resolution Report to Product Teams, VEMO, MACs)

End Event:

Escalated issue resolved and documented.

Potential KPIs/Measures:

  • Time to resolution for escalated issues.
  • Reduction in recurring escalations.
  • Satisfaction of Product Teams with resolution process.
  • Escalation frequency.

Need for standardized vendor decision-making criteria identified.

Process Flow 8: Strategic Vendor Review Meeting Process

Objective:

Manage strategic and critical vendor relationships to sustain key partnerships.

BPMN Diagram:

Swimlanes & Partners:

  • VEMO: Lead, Facilitator, Coordinator (A/R per document).
  • Product Teams: Stakeholder, Information Provider.
  • ITOS/SAM: Stakeholder, Information Provider.
  • Key Vendor (External): Participant.

Start Event:

Scheduled review period (e.g., quarterly) or ad-hoc need for strategic discussion.

Process Steps & Flows:

Swimlane: VEMO (Responsibility: Manage strategic vendor relationships)

  • Task: Plan and schedule strategic vendor meeting.
  • Task: Prepare agenda, gather internal input. (Output: Meeting Agenda, Internal Briefing Document; Output Message Flow: Invitation & Agenda to attendees)
  • Task: Conduct/Facilitate meeting.

Swimlane: Product Teams

  • Task: Provide input on vendor’s operational performance.
  • Task: Attend meeting as key internal stakeholder.

Swimlane: ITOS/SAM

  • Task: Provide input on technical performance, software/asset issues.
  • Task: Attend meeting if relevant.

Swimlane: Key Vendor (External)

  • Input Message Flow: Meeting Invitation & Agenda.
  • Task: Prepare for and attend meeting.
  • Task: Present performance updates, address concerns, discuss opportunities.

Swimlane: VEMO

  • Task: Document meeting outcomes, action items. (Output: Meeting Minutes, Action Item List)
  • Task: Disseminate meeting outcomes to stakeholders. (Output Message Flow)
  • Task: Follow up on action items.

End Event:

Vendor review meeting concluded, actions documented and tracked.

Potential KPIs/Measures:

  • Regularity of strategic vendor meetings.
  • Completion rate of action items.
  • Stakeholder satisfaction with meeting effectiveness.
  • Measurable improvements in vendor relationship health.
  • Vendor-driven innovation outcomes.

Process Flow 9: Vendor Performance Management Process

Objective:

Ensure vendors meet performance expectations and drive value.

BPMN Diagram:

Swimlanes & Partners:

  • VEMO: Lead, Analyst, Improvement Driver (A/R per document).
  • Product Teams: Information Provider, Stakeholder.
  • Key Vendor (External): Data Provider, Participant.

Start Event:

New vendor contract with defined SLAs/KPIs or scheduled performance review cycle.

Process Steps & Flows:

Swimlane: VEMO (Responsibility: Define and monitor KPIs/SLAs; drive performance improvements)

  • Task: Define/confirm KPIs and SLAs for vendors. (Output: Vendor KPIs/SLAs Document)
  • Task: Monitor vendor performance against agreed metrics. (Input: Vendor performance reports, operational feedback)
  • Task: Collect and analyze performance data.
  • Task: Report on vendor performance. (Output: Vendor Performance Report; Output Message Flow to internal stakeholders)
  • Task: Identify performance issues/improvement opportunities.
  • Task: Drive vendor performance improvement initiatives. (Output Message Flow: Performance Improvement Plan/Request to Vendor)

Swimlane: Product Teams

  • Task: Provide operational performance feedback. (Output Information Shared to VEMO)
  • Task: Participate in performance improvement discussions.

Swimlane: Key Vendor (External)

  • Task: Provide performance data as per contract.
  • Task: Participate in performance reviews and implement improvement actions.

Swimlane: VEMO

  • Task: Disseminate performance reports and improvement outcomes to stakeholders. (Output Message Flow)

End Event:

Vendor performance reviewed, reported, and improvement actions implemented/tracked (cyclical).

Potential KPIs/Measures:

  • Vendor adherence to SLAs/KPIs.
  • Number of performance improvement initiatives and success rate.
  • Value realization from vendor engagements.
  • Vendor performance trend improvement.

Process Flow 10: Software/Infrastructure Asset License & Contract Management

Objective:

Manage software assets, licenses, and infrastructure assets effectively, ensuring compliance and optimizing spend.

BPMN Diagram:

Swimlanes & Partners:

  • ITOS/SAM: Lead, Technical Authority, Operational Manager (A/R per RACI).
  • Product Teams/End Users: Requester, User.
  • MACs: Procurement Executor (Consulted, if applicable).
  • VEMO: Informed (if applicable for commercial vendors).

Start Event:

Software/infrastructure need identified, license renewal upcoming, or audit scheduled.

Process Steps & Flows:

Swimlane: Product Teams/End Users

  • Task: Submit software/infrastructure requests. (Output: Software/Infrastructure Request)

Swimlane: ITOS/SAM (Responsibility: Oversee software asset management, etc.)

  • Input: Software/Infrastructure Request.
  • Task: Receive and triage requests, track licenses/entitlements, manage renewals/upgrades, conduct audits, manage infrastructure assets and related contracts.
  • Task: Initiate proactive software audits or license renewals as scheduled.
  • Task (Gateway): New procurement or significant contract action needed?
    • Yes: Initiate procurement/contract action via MACs. (Output Message Flow: Procurement/Contract Request to MACs)
    • No: Fulfill request from existing licenses/assets or manage renewal directly.

Swimlane: MACs (Responsibility: Execute procurement if requested)

  • Input Message Flow (if applicable): Procurement/Contract Request.
  • Task: Execute procurement/contracting process (as per Flow 4).

Swimlane: VEMO (Responsibility: Informed on significant commercial vendor software/asset matters)

  • Input (Implied): Updates from ITOS/SAM on commercial vendor activities.

Swimlane: ITOS/SAM

  • Task: Disseminate outcomes (e.g., fulfilled requests, audit results) to stakeholders. (Output Message Flow to Product Teams, VEMO if applicable)

End Event:

Request fulfilled; license/asset/contract managed; compliance ensured (cyclical for renewals/audits).

Potential KPIs/Measures:

  • License compliance rate.
  • Cost savings through software optimization.
  • Timeliness of request fulfillment.
  • Accuracy of asset inventory.
  • Audit frequency.
